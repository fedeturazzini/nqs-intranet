01 — ¿Qué hace?

De render a maqueta arquitectónica en segundos

Maquette toma una imagen de arquitectura — render, aerial, foto — y la transforma en una maqueta física virtual: ese modelo de mesa que se usa en presentaciones de proyecto, fotografiado sobre fondo blanco.

El estilo lo controlás con una imagen de referencia. La máquina vive 100% en Weavy y genera 5 variaciones en paralelo.

¿Para qué sirve? Como es difícil hacer maquetas físicas hoy en día, esto te da la misma sensación de presentación. Podés combinarlo con InMotion para animar o hacer un film del proyecto.

02 — Cómo arrancar

Copiar la máquina al espacio de trabajo

Como en todas las herramientas: nunca se trabaja sobre la máquina original. Se copia al espacio del proyecto.

1
En Weavy, andá a My Files → MACHINES
2
Buscá la carpeta MAQUETTE
3
Copiá todo el contenido al lugar que te corresponde (carpeta del proyecto nomenclada)
4
Trabajá sobre la copia. Nunca sobre la máquina original.

03 — Anatomía

Las partes de la máquina

1
Imagen Base — el render o foto del proyecto. Lo único que siempre cambiás.
2
Referencia Maqueta — foto de una maqueta real que define el estilo. El validado por el AD.
3
System Prompt — instrucciones del LLM. No se toca salvo ajuste en Claude.
4
Any LLM — analiza las dos imágenes y genera prompts. Solo corrés Run Model.
5
Array — distribuye los prompts a los nodos. No se modifica.
6
Resolución + Aspect Ratio — 2K exploración · 4K final · Auto para proporciones libres.
7
Nanobanana × 5 — los 5 nodos Gemini que generan en paralelo. Seleccionás todos → Run Selected.

04 — Proceso

Correr la máquina paso a paso

1
Cargá la imagen base — el render o foto del proyecto
2
Verificá la referencia de maqueta — si querés cambiar el estilo, cargá una nueva (previa charla con el AD)
3
Run Model en el LLM — regenera los prompts. Siempre primero si cambiaste alguna imagen.
4
Configurá Resolución y Aspect Ratio — 2K / auto para exploración
5
Seleccioná todos los Nanobanana → Run Selected — los 5 generan en paralelo
6
Si algo no convence, volvé a correr Run Model en el LLM y corrés de nuevo
Regla de oro Cada vez que cambiás la imagen base o la referencia, corré Run Model primero. Si no, los prompts siguen siendo los anteriores.

05 — Errores y soluciones

Problemas típicos y cómo resolverlos

Contaminación de arquitectura — La imagen de referencia "contamina" la base: aparecen torres o elementos de la referencia en el resultado.
Solución: volvé a correr Run Model en el LLM para que regenere un prompt diferente.
Variaciones muy similares entre sí — Los 5 outputs se parecen demasiado.
Solución: corrés Run Model en el LLM de nuevo — el prompt cambia y da más diversidad.
El estilo no es el esperado — La maqueta no tiene el look correcto.
Solución: cambiá la imagen de referencia por una que muestre mejor el material o estilo que buscás → Run Model → Run Selected.
Tip Para generar nuevas variaciones de una misma imagen ya cargada, siempre corré Run Model en el LLM de nuevo. No alcanza con correr solo los Nanobanana.

06 — Avanzado

Modificar el System Prompt con Claude

El System Prompt que viene con la máquina ya está armado para funcionar bien en la mayoría de los casos. Esto es una excepción — deberían ser pocos los proyectos que lo necesiten. Pero si querés algo muy específico que la imagen de referencia sola no logra, podés abrir un chat cualquiera en Claude (no hace falta ningún proyecto especial), pegarle el prompt y decirle: "Tengo este system prompt, necesito modificarlo para que cuando vea una imagen haga X". Ahí le das una vueltita de tuerca.

1
Copiá el System Prompt del nodo en Weavy
2
Pegalo en Claude y pedile algo específico — ej: "en zonas con agua, usar material acrílico"
3
Reemplazá el System Prompt en Weavy con lo que te devuelve Claude
4
Run Model en el LLM → Run Selected en los Nanobanana
Sistematizar el look Una vez que encontrás el prompt que da el resultado buscado, lo guardás y lo reutilizás en todos los proyectos con ese estilo.
Importante Leé el System Prompt modificado antes de correr — Claude no tiene contexto del modelo generativo.
