01 — Cómo arrancar

Copiar la máquina al espacio de trabajo

Como hicimos en las otras presentaciones: dentro de Weavy, vamos a My Files → MACHINES, buscamos MOCK UP y la copiamos a nuestro espacio de trabajo en la carpeta del proyecto que estamos atacando.

Ruta Weavy · My Files → MACHINES → MOCK UP → copiar al proyecto
Weavy · MACHINES / MOCK UP 01 / 04
02 — Dos variantes

Una máquina, dos caminos

El canvas trae dos workflows listos. Copiás el que necesitás según lo que te aporte el cliente.

1
Render del espacio
El cliente te pasa la imagen del espacio (render o foto) y de ahí sacamos los materiales.
2
Recortes de materiales
El cliente no te pasa el espacio, solo capturas sueltas de los materiales que lo van a componer.
En base a eso elegís cuál de las dos te copiás y pegás en el proyecto.

03 — Partes de la máquina

Las partes de la máquina

1
System Prompt
Las instrucciones fijas que recibe el LLM. No se modifica.
2
Any LLM
Analiza las dos imágenes y arma el prompt de generación. Tampoco se modifica.
3
Imagen madre
Referencia de cómo queremos que sea el moodboard final. Se puede cambiar.
4
Render del cliente
La imagen del espacio del proyecto. Acá va lo que aporta el cliente.
5
Nano Banana
El nodo Gemini 3.1 Flash que genera el moodboard final.

04 — Correr la máquina

Run Model y configurar Nano Banana

El System Prompt y el LLM no se tocan. Solo cambiás las imágenes y corrés en orden:

1
Run Model en el LLM
Se actualiza el prompt de generación con las nuevas imágenes.
2
Run Model en Nano Banana
Genera el moodboard final.
Configuración del nodo Nano Banana Apretás el nodo y seteás resolución 2K para exploración / 4K para final, la proporción que quieras, y dejás siempre activado el Web Search.
Después de generar Como siempre, renombramos las imágenes siguiendo la lógica que ya conocemos de las otras presentaciones, las subimos a Miro y pimponeamos con el AD. Una vez OK, le pedimos al PM la carpeta para subirlo, o lo descarga directo desde Miro al ser imágenes cortas.
Siguiente paso ¿Seguimos? Podés usar In Motion para animar el mockup generado, o Reframe para generar variaciones del mockup en distintos ángulos.
